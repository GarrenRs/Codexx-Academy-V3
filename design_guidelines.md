# إرشادات التصميم - منصة النادي التدريبي الإلكتروني

## نهج التصميم (Design Approach)

**المرجع**: تصميم يجمع بين احترافية **Discord** (للدردشة والغرف) + **Linear** (للمشاريع والمهام) + **Notion** (للتنظيم) مع لمسة نادي إلكتروني عصري.

**المبادئ الأساسية**:
- تجربة نادي رقمي احترافي وسلس
- تنظيم واضح للتسلسل الهرمي (Admin → Manager → Leader → Member)
- تفاعلية سلسة للتعاون الجماعي
- اتجاه RTL كامل مع خط Cairo

## العناصر الأساسية

### A) الطباعة (Typography)
```
الخط الرئيسي: Cairo (Google Fonts)
- العناوين الكبيرة: font-bold text-2xl/3xl/4xl
- العناوين الفرعية: font-semibold text-lg/xl
- النصوص الأساسية: font-normal text-base
- النصوص الثانوية: font-normal text-sm
- التسميات الصغيرة: font-medium text-xs

التسلسل الهرمي:
- صفحات الهبوط: Cairo Bold (700) للعناوين الرئيسية
- لوحات التحكم: Cairo SemiBold (600) للعناوين
- المحتوى: Cairo Regular (400)
- الأزرار والتسميات: Cairo Medium (500)
```

### B) نظام التباعد (Spacing System)
**وحدات Tailwind المستخدمة**: 2, 3, 4, 6, 8, 12, 16, 20, 24

```
- فواصل المكونات الصغيرة: p-2, gap-3
- فواصل الكروت والعناصر: p-4, p-6, gap-4
- فواصل الأقسام الداخلية: p-8, py-12
- فواصل الأقسام الكبيرة: py-16, py-20, py-24
- الهوامش الخارجية: mx-4, mx-6, mx-auto
- التباعد بين العناصر: space-y-4, space-y-6, space-y-8
```

### C) مكتبة المكونات (Component Library)

#### 1. التنقل والهيكل
- **Sidebar النادي**: جانبي ثابت (fixed) يحتوي:
  - شعار النادي في الأعلى
  - قائمة الغرف مع أيقونات
  - قائمة المجموعات النشطة
  - إشعارات فورية
  - بروفايل المستخدم في الأسفل
  - عرض: w-64 على الشاشات الكبيرة، drawer قابل للطي على الموبايل

- **Header Bar**: شريط علوي يحتوي:
  - عنوان الصفحة الحالية
  - breadcrumbs للمسار
  - أزرار الإجراءات السريعة
  - زر الإشعارات
  - بحث سريع

#### 2. كروت العرض
- **RoomCard**: كرت غرفة التخصص
  - أيقونة التخصص
  - اسم الغرفة
  - عدد المجموعات والأعضاء
  - مدير الغرفة
  - badge للحالة (نشطة/قيد العمل)
  - زر "دخول الغرفة"

- **GroupCard**: كرت المجموعة (4 أعضاء كحد أقصى)
  - أفاتار الأعضاء (4 دوائر متداخلة)
  - اسم المجموعة
  - قائد المجموعة
  - المشروع الحالي
  - progress bar للتقدم
  - أزرار: انضمام/مغادرة/دخول

- **ProjectCard**: كرت المشروع
  - عنوان المشروع
  - وصف مختصر
  - صاحب الاقتراح
  - عدد الأصوات مع voting meter
  - الحالة: مقترح/قيد التصويت/معتمد/قيد التنفيذ
  - أزرار الإجراءات حسب الصلاحية

- **TaskCard**: كرت المهمة
  - عنوان المهمة
  - الوصف
  - المجموعة/الغرفة المسؤولة
  - الأولوية (عالية/متوسطة/منخفضة)
  - التقدم المئوي
  - تاريخ الاستحقاق
  - checkbox للإنجاز

#### 3. لوحات التفاعل
- **VotingPanel**: لوحة التصويت (للمديرين فقط)
  - عرض تفاصيل المشروع
  - قائمة المصوتين
  - نتائج التصويت بـ progress bars
  - أزرار: موافق/رفض (للمديرين)
  - عداد الوقت المتبقي للتصويت

- **TaskBoard**: لوحة المهام (نمط Kanban)
  - أعمدة: To Do, In Progress, Review, Done
  - drag & drop للمهام
  - filters حسب الأولوية/المجموعة
  - إضافة مهمة سريعة

- **ChatBox**: صندوق الدردشة
  - قائمة الرسائل مع scroll تلقائي
  - أفاتار المرسل على اليمين (RTL)
  - timestamps
  - indicators للأعضاء المتصلين
  - input مع emoji picker
  - زر إرسال
  - typing indicators

#### 4. النماذج
- **تسجيل الدخول/التسجيل**: 
  - تصميم مركزي بسيط
  - حقول كبيرة وواضحة
  - تكامل Replit Auth
  - روابط التبديل بين الصفحات

- **نماذج الإنشاء** (غرفة/مجموعة/مشروع/مهمة):
  - modal أو صفحة مخصصة
  - حقول منظمة بـ sections
  - validation واضحة
  - أزرار إجراء بارزة

#### 5. لوحات التحكم الإدارية
- **Admin Dashboard**: 
  - إحصائيات عامة (cards مع أرقام كبيرة)
  - رسوم بيانية للنشاط
  - قائمة الغرف مع quick actions
  - المشاريع المعلقة
  - طلبات الانضمام

- **Room Manager Dashboard**:
  - نظرة عامة على الغرفة
  - المجموعات النشطة
  - توزيع المهام
  - تقييمات الأداء

### D) الحركات (Animations)
استخدام محدود جداً:
- **hover**: scale-105 للكروت، opacity-80 للأزرار
- **transitions**: duration-200 للتفاعلات السريعة
- **loading**: spinner بسيط أو skeleton screens
- **notifications**: slide-in من الأعلى
- **modal/drawer**: fade + slide

## البنية العامة للصفحات

### صفحة الهبوط (Landing)
- Hero section: عنوان قوي + وصف + CTA + صورة توضيحية
- Features: 3-4 كروت تشرح المميزات
- How it Works: 4 خطوات بـ timeline
- Testimonials: شهادات المتدربين السابقين
- CTA النهائي

### صفحات التطبيق
- Layout ثابت: Sidebar + Header + Content Area
- Content Area: responsive grid (1 col mobile, 2-3 cols desktop)

## الصور (Images)

### صفحة الهبوط:
1. **Hero Image**: صورة توضيحية لفريق يعمل معاً (illustration style)، توضع على يسار النص (RTL)
2. **Features Icons**: أيقونات مخصصة لكل ميزة (من Heroicons)
3. **How it Works**: أيقونات مرقمة للخطوات

### داخل التطبيق:
- **أفاتار المستخدمين**: دوائر مع initials للأسماء
- **أيقونات الغرف**: أيقونات تمثل التخصصات (كود، تصميم، تسويق، إلخ)
- **Empty States**: رسومات بسيطة عند عدم وجود محتوى

## الاستجابة (Responsiveness)

```
Mobile (< 768px):
- Sidebar يتحول إلى drawer
- الكروت بعرض كامل (1 col)
- Header مبسط
- نصوص أصغر قليلاً

Tablet (768px - 1024px):
- 2 columns للكروت
- Sidebar ثابت أو قابل للطي

Desktop (> 1024px):
- 3-4 columns حسب الصفحة
- Sidebar ثابت دائماً
- استخدام كامل للمساحة
```

## ملاحظات RTL خاصة
- جميع الهوامش والتباعدات معكوسة تلقائياً
- الأيقونات توضع على اليمين
- القوائم المنسدلة تفتح من اليمين
- الـ Sidebar على الجانب الأيمن
- القوائم المرقمة من اليمين لليسار
- الـ Chat bubbles: المستخدم على اليمين، الآخرون على اليسار
